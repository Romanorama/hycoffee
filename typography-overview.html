Normalized Groups

These were the groups with the clearest near-duplicates. They now point at named `--type-*` tokens in `global.css`.

Captions and labels

Current spread: 10px, 11px, 12px, 13px, 15px, 20.9px, plus several tiny clamps. The 20.9px origins caption is the largest outlier.

Body copy

Current spread: 14px, 15px, 16px, 17px, 18px, 23px, plus 14-17px and 15-18px clamps. This wants two or three body roles.

Buttons

Global buttons exist, but local buttons use 12-14px, 14-16px, 16-22px, 18px, and 20px. Good fit for button-sm, button-md, button-lg.

Section headings

The 34-36px / 60-62px heading range appears several times. It likely only needs two roles: section and large-section.

Kaffee table

The coffee page has a coherent micro-scale, but it is detached from global tokens. Better to create table/detail tokens than force generic paragraphs.

Exact token matches

There are 21 local declarations that already match global sizes. These are low-risk replacements or class applications.

Outliers to Decide

These are not automatically wrong. They just deserve an explicit decision before normalization.

  • 20.9pxOrigins caption: unusually large for an uppercase label.
  • 18.45pxScience link focus state: changes size on focus, likely worth avoiding.
  • 23.45pxMax value of the same focus state, very close to the normal 23px max.
  • 30px / 22pxOrigins accordion and selector scale. Coherent locally, but separate from global heading/body scale.
  • 10pxCoffee page version toggle. Looks like a dev/control label rather than product typography.
  • 36pxAbout page contact/social text and mobile nav links. Similar size, different roles.
