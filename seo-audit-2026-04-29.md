# SEO Metadata Audit — HyCoffee

_Date: 2026-04-29_

## Foundation: solid

[BaseLayout.astro](src/layouts/BaseLayout.astro) is well-built. It centralizes title, description, canonical, robots, OG/Twitter tags, hreflang (de/en/x-default), JSON-LD, OG image generation (1200×630 jpg), article timestamps, and noindex. Per-page usage is consistent — every page passes `title`, `description`, `ogImage`, and `ogImageAlt`. [seo.ts](src/lib/seo.ts) injects `Organization` + `WebSite` schema globally; [sitemap.xml.ts](src/pages/sitemap.xml.ts) and [robots.txt.ts](src/pages/robots.txt.ts) are wired up; `astro.config.mjs` has `site` set.

## Findings, ranked

### 1. German descriptions strip umlauts — [seo.ts:6](src/lib/seo.ts#L6) and most pages
Examples: `Spezialitaetenkaffees`, `fuer`, `widerstandsfaehigere`, `Geschaeftsbedingungen`, `Hintergruende`, `Wertschoepfung`, `ueber`. UTF-8 is fine in meta tags — these read poorly in German SERPs and hurt CTR. Titles like `Über Uns` already use umlauts, so this is inconsistent on the same site.

**Highest-impact fix.** Files to touch:
- `src/lib/seo.ts` (`DEFAULT_DESCRIPTION`)
- `src/pages/index.astro`
- `src/pages/origins.astro`
- `src/pages/agb.astro`
- `src/pages/wissen/index.astro`
- `src/pages/ueber-uns.astro` (verify)
- `src/pages/datenschutz.astro` (verify)

### 2. Title format inconsistent — [index.astro:42](src/pages/index.astro#L42), [en/index.astro:44](src/pages/en/index.astro#L44)
Homepages: `HyCoffee – Klimaresilienter Rohkaffee` (brand first). Every other page: `Topic – HyCoffee` (topic first). Best practice is topic first for SERP truncation. Either flip the homepage or accept the inconsistency, but pick one.

### 3. Articles never set `modifiedTime` / `dateModified` — [wissen/[slug].astro:34-46](src/pages/wissen/%5Bslug%5D.astro#L34-L46)
`BaseLayout` accepts `modifiedTime` but no page passes it; the `Article` JSON-LD has `datePublished` but no `dateModified` and no `author`. Add an optional `updatedAt` to the wissen schema → pass through to both. Author defaulting to the Organization is fine.

Same applies to [en/knowledge/[slug].astro](src/pages/en/knowledge/%5Bslug%5D.astro).

### 4. Sitemap has no `<lastmod>` — [sitemap.xml.ts](src/pages/sitemap.xml.ts)
Only `<loc>`. Adding `<lastmod>` (from `publishedAt`/git mtime for static pages) helps crawl prioritization. Low effort.

### 5. OG image lacks width/height — [BaseLayout.astro:110-113](src/layouts/BaseLayout.astro#L110-L113)
`getImage` returns 1200×630, but `og:image:width` / `og:image:height` aren't emitted. Some scrapers (LinkedIn especially) prefer these. Two extra lines.

### 6. Hreflang for legal pages points EN → DE page — [i18n.ts:16-18](src/lib/i18n.ts#L16-L18)
`/agb`, `/datenschutz`, `/impressum` map to themselves for EN. Pages are `noindex`, so this is mostly cosmetic, but technically `hreflang="en"` should not be emitted for pages that have no English version. Cleanest: skip alternate links when the page is `noindex` or has no real translation.

### 7. Some titles too generic — [origins.astro:44](src/pages/origins.astro#L44), [wissen/index.astro:15](src/pages/wissen/index.astro#L15)
`Origins – HyCoffee` (18 chars), `Wissen – HyCoffee` (17 chars). Plenty of room to add a keyword phrase, e.g. `Origins – Klimaresiliente Kaffeeherkunft in Uganda – HyCoffee`.

### 8. `kaffee.astro` description = `pageIntro` — [kaffee.astro:25](src/pages/kaffee.astro#L25)
Whatever editors type in Keystatic becomes the meta description verbatim. No length guard. Worth either a dedicated `seoDescription` field in the singleton, or trimming/clamping at build.

### 9. No `BreadcrumbList` schema on articles
For `/wissen/[slug]`, a `BreadcrumbList` (Home → Wissen → Article) is a small, well-supported add that improves rich-result eligibility.

### 10. `DEFAULT_DESCRIPTION_EN` not wired up
`DEFAULT_DESCRIPTION_EN` exists in [seo.ts:7](src/lib/seo.ts#L7) but [BaseLayout.astro:31](src/layouts/BaseLayout.astro#L31) defaults to the German one regardless of `lang`. Every page passes a description today, so it's dormant — but if anyone forgets a description on an EN page, they'll get German.

## Suggested order of attack

1. Fix umlauts in all German descriptions + `seo.ts` defaults (~15 min, biggest visible win).
2. Decide title format and align homepage (~5 min).
3. Add `modifiedTime` / `dateModified` / `author` to articles + `<lastmod>` in sitemap (~30 min).
4. Add OG image dimensions + EN default fallback wired up (~10 min).
5. `BreadcrumbList` schema for articles (~15 min).
